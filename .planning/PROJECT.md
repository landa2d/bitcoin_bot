# AgentPulse

## What This Is

A multi-agent intelligence platform for the AI agent economy: eight cooperating Docker services (Telegram-facing bot, conversational middleware, background processor, analyst, newsletter writer, research agent, LLM proxy, web frontend) that ingest content, synthesize findings, and publish daily-to-weekly outputs (newsletters, X posts, briefings) for the operator. Controlled via Telegram commands; deployed on a single Linux server with Supabase as the shared data store.

## Core Value

**Synthesis with editorial integrity.** Autonomous ingestion and drafting accelerate the operator's output, but every consequential publication is gated by human approval — silence and homogenization are the failure modes to design against.

## Current Milestone: v2.3 Pre-Publish Evaluation Step

**Goal:** Insert an automated, two-layer evaluation step between newsletter draft generation and publish — a hard deterministic gate that catches fabrication/mechanical errors (hold + escalate, never rewrite) and an LLM-judge + bounded feedback-rewrite loop that catches voice/editorial misses — so these errors are caught (and where safe, auto-corrected) before the operator's Monday review. **Passing the eval does NOT auto-publish; the existing human gate is unchanged.**

**Architecture (reconciled against the live system — see `.planning/research/INVENTORY.md`):** Both layers run in the **newsletter service** at the two generation save points, because the true source fact base (`input_data` for single-pass, `blocks_data` for block_v1) exists in-memory only there and is unrecoverable at publish time. The `newsletter_poller` is the dumb sequencer: it calls the deterministic gate, then the standalone Layer-2 eval module, and acts on returned verdicts. The literal **Processor stays a dumb sequencer** — it triggers generation, owns the publish gate, and surfaces eval results in the Friday notify via a plain select. **No LLM calls and no retry/rewrite state in the Processor.**

**Target features (phase shape):**
- **Continuity + exemplar loader (R4, Phase 1):** `load_edition_context()` — prior published editions' angles + operator-approved exemplar paragraphs, fail-loud-but-not-fatal on empty; resurrects the dead Phase E voice check and feeds the judge.
- **Layer 1 — deterministic gate (no LLM):** fabrication checks (live GitHub repo + star-count, URL liveness HEAD, arXiv-ID validation, named-study-vs-fact-base, entity-merge / verbatim-in-a-single-source) + mechanical editorial regexes (H1/title echo in body, reading-mode label leak, recycled closer vs prior edition, stat duplicated verbatim from prior edition). Runs on every edition, short-circuits before any LLM. **Any fabrication flag → `held` + `do_not_publish` + reason + Gato escalation, never the rewrite loop.**
- **Layer 2 — LLM judge + feedback-rewrite loop (standalone module):** Sonnet judge scores each version 1–5 across exemplar-anchored dimensions (cross-edition continuity bridge, hedging filler, clickbait/fear hooks vs professor voice, repeated sub-topics, specificity) with per-dimension pass thresholds; on failure, re-call the writer with structured, specific feedback; re-eval; **N=2 attempts max, no best-effort publish**. Pass → normal human gate; fail after N → real `held` + Gato escalation.
- **Persistence (`edition_evals`, migration 045):** one row per draft per attempt — edition id/number, attempt, layer, deterministic flags (jsonb), per-dimension judge scores (jsonb), judge feedback (text), verdict (`passed`/`held_fabrication`/`held_voice`/`escalated`). SQL-first, operator-applied via MCP after DDL review. Fail-loud: eval error → `eval_status='error'`, never a silent zero.
- **Governance + surfacing:** governed `edition_eval` agent via `llm-proxy:8200` (own wallet, `allow_negative=false`, hard cap, `on_cap_behavior='reject'`); reuse `send_telegram` for hold/escalation alerts (hardened against silent no-op); `/newsletter_eval` Gato command (+ `inject-gato-brain.mjs` allowlist).

**Key reconciled decisions (operator-confirmed 2026-06-22):**
- Eval runs in the newsletter service at generation time (NOT literally in the Processor) — the only place the true fact base lives; honors "Processor stays a dumb sequencer, no LLM/retry-state."
- R4 (continuity + exemplar loader) folded in as Phase 1 — a hard dependency of the Layer-2 continuity/voice judge.
- The milestone command is authoritative; audit specs 01/07 are the implementation reference (wiring points, `verify_draft` reuse, governed-agent seeding pattern, fail-loud rules). Sonnet model = `claude-sonnet-4-6` (spec 01's `claude-sonnet-4-20250514` is EOL).

**Architectural rules (do not violate):** Hardcoded orchestration over LLM reasoning in control flow; autonomy where errors are cheap/reversible, human gates where expensive/silent; fail loud (NULL ≠ intent, no bare excepts); diagnostic-before-fix (inventory done → plan → operator approval → implement). All LLM via `llm-proxy:8200`; no supabase-py `.in_()` (direct PostgREST + `Accept-Profile` where schema-isolated); SQL-first, operator-applied; no Docker/unrelated-agent changes beyond the wiring the spec names.

## Constraints

- **Architecture**: Single Linux server, Docker Compose orchestration, Supabase as shared store — no Kubernetes, no message broker, no external cache. New work integrates here.
- **LLM access**: All model calls route through `http://llm-proxy:8200`. Direct provider SDK calls are forbidden (budget governance + the RivalScope lesson).
- **Schema access**: `economy_map` tables accessed via direct PostgREST HTTP with `Accept-Profile: economy_map` header. Do not use supabase-py `.in_()` (silent failure).
- **Frontend-only (v2.0)**: No backend, pipeline, Supabase, or content/data changes. Dual-mode *content* logic is unchanged — only the mode toggle's placement and styling move.
- **One light-mode accent system (v2.0)**: A single violet accent across all sections (links, active tab, card borders, dots); dark mode deferred. Reuse the existing publish path; the mockup is a reference for intent, not markup to copy.
- **Publish path**: Public-site changes ship via the existing `aiagentspulse.com` scoped web rebuild (single `agentpulse-web` container) confirmed in v1.0 Phase 0 — no new infra.
- **Editorial framing in human hands**: `live_tension` and synthesis prompt voice (via `synth_identity.md`) stay operator-controlled; the loop synthesizes, the operator frames.
