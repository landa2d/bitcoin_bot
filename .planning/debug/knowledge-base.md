## single-pass-writer-empty — single-pass newsletter writer parsed claude-sonnet-4-6 large response as "empty"
- **Date:** 2026-06-24
- **Error patterns:** Failed to parse model response as JSON, Expecting value: line 1 column 1 (char 0), content[0].text empty, newsletter generation fails, claude-sonnet-4-6, write_newsletter, JSONDecodeError
- **Root cause:** Brittle JSON extraction — `response.content[0].text.strip()`, strip markdown fences ONLY when `text.startswith("```")`, then `json.loads()`. claude-sonnet-4-6 frames large single-shot JSON output stochastically (bare / ```json-fenced / occasionally prose-prefixed); any framing that wasn't bare-or-leading-fence skipped the strip and `json.loads` failed at "char 0". NOT empty content, NOT a non-text/thinking block, NOT multi-block splitting — verified by inspecting 7 real responses (always one fully-populated text block). The "char 0" JSONDecodeError fires for ANY non-JSON leading char, not just empty; a non-text block would raise AttributeError instead.
- **Fix:** Shared `response_text()` (joins text blocks) + `parse_llm_json()` (raw → ```json/``` fenced [case-insensitive, DOTALL] → first balanced `{...}` [string/escape-aware]; FAIL LOUD with logged head/tail when nothing parses — never silent-empty). Applied at qualitative_review, editorial_prepass, and the writer (both branches); strategic_editor uses response_text() for multi-block safety.
- **Files changed:** docker/newsletter/newsletter_poller.py, tests/test_27_llm_json_extract.py

## eval-tier1-entity-fp — deterministic gate tier1_entity extractor flagged 27 noise "fabrications" on first real single-pass draft (edition #104)
- **Date:** 2026-07-03
- **Error patterns:** tier1_entity, held_fabrication, deterministic gate, verify_draft, verification.py, run_deterministic_gate, edition_evals, fabrication flags, sentence-initial capitalized words, sentence fragments flagged as entities, entity_merge noise, false positive entities, quote extraction, single-pass draft, proper noun regex, stop words
- **Root cause:** TWO independent extractor defects in docker/newsletter/verification.py, both amplified by a whitespace-collapse (`\s+`→single space in _extract_claims_from_prose) that erased the sentence/line boundaries the regexes needed. (A) Boundary-blind capitalized-word regexes: `_PROPER_NOUN_SINGLE`/`_MULTI_CAP` treat grammatical (sentence/clause/line-initial) capitalization as a named entity and rely ENTIRELY on an unwinnable `_STOP_WORDS` list to reject non-entities (the regex comment claimed "not at sentence start" but never enforced it) → "According", "Nobody", "Timeline", "What Zuckerberg" etc. flagged as tier-1 fabrications. (B) Straight-quote parity flip in `_QUOTED`: identical ASCII `"` chars with a `{10,}` inline content minimum — a sub-10-char quote ("harness.") fails to consume its opening quote, shifting odd/even quote parity so the regex captures the UNQUOTED inter-quote narrative as 498/2048/63-char fake "quotes" (truncated to 80) → whole sentence fragments flagged; plus rhetorical/term/scare quotes with no verifiable token graded as fabrications. SIDE NOTE — the version-attribution "puzzle" was NOT a bug: every flag's version label matched the body it was extracted from; the confusion was a case-sensitive grep of "AUDIENCE:" missing "Target audience:" plus a transcription slip on "Amazon Web Services" (correctly labeled impact). arXiv membership + duplicated_stat checks were never defective and their signal was preserved.
- **Fix:** (verification.py) Fix A — `_is_boundary_initial()` on a newline-preserving `clean_nl` buffer; `_PROPER_NOUN_SINGLE` skips boundary-initial single words, `_MULTI_CAP` skips sentence/line-opening phrases whose first word is a stop word. Fix B — positional quote pairing (`"([^"]*)"`, length filtered AFTER matching) + `_quote_has_verifiable_token()` (number / proper noun / 3+-char acronym required) so rhetorical quotes aren't fabrications. Fix C — one targeted stop-word add (`devops`). No change to _check_arxiv_membership, _check_cross_edition (duplicated_stat), or _build_block_list — edition #104 fabrication 31→11 (tier1_entity ×8 genuine, arxiv ×2, entity_merge ×1 Google Cloud), verified live in-container against the frozen golden fixture (tests/test_28_edition_104_golden.py).
- **Files changed:** docker/newsletter/verification.py, tests/fixtures/edition_104_gate_golden.json, tests/test_28_edition_104_golden.py
