<objective>
Build the three reusable contracts the intake classifier poller (Plan 02) and the verification tests (Plan 03) depend on, so that those plans implement and test against fixed interfaces rather than inventing them:

1. A config-driven confidence floor (default 0.6) in `config/agentpulse-config.json` (D-06).
2. A pair of `economy_map` PostgREST helpers in the processor — one INSERT (write), one existence-check SELECT (read) — because no Python `economy_map` write analog exists in the tree.
3. A proxy-routed DeepSeek classifier function + its prompt constant that returns `block_slug` + `tag_confidence` per event (INTK-02), routed through `http://llm-proxy:8200` (NOT the SDK-direct `routed_llm_call`).

Purpose: ROADMAP success criterion 2 mandates the classifier call route through `http://llm-proxy:8200` with no direct DeepSeek SDK call, verified by proxy-side evidence per classification. The in-tree `routed_llm_call()` calls provider SDK clients directly and therefore cannot satisfy this — this plan builds the proxy HTTP path instead. CLAUDE.md/PROJECT.md forbid direct provider SDK calls (the RivalScope lesson).

Output: New config section, two PostgREST helper functions, one prompt constant, one classifier function — all in `docker/processor/agentpulse_processor.py` (and config). No poller, no schedule registration, no INSERT loop yet (Plan 02).
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| scraped newsletter event text → DeepSeek classifier | event `title`/`summary` originate from third-party scraped sources (RSS/HN/X); untrusted content crosses into an LLM prompt |
| processor → llm-proxy (:8200) | the processor's per-agent proxy key crosses here; the proxy enforces auth, budget, rate limit |
| processor (service_role) → economy_map.timeline_entries | service_role key bypasses RLS by design; write surface must be tightly scoped |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Tampering / Elevation | INTAKE_CLASSIFIER_PROMPT (untrusted event text) | mitigate | Prompt frames event text as untrusted data to classify, never as instructions; output schema is constrained to exactly two keys (`block_slug` ∈ allowed set, `tag_confidence` float); Plan 02 validates the returned slug against the seven-slug allow-list before any DB write, so prompt-injection cannot widen the write target beyond the allowed slugs or 'unsorted' |
| T-05-02 | Elevation of Privilege / Tampering | economy_map service_role PostgREST helpers | mitigate | Helpers are scoped to ONLY `timeline_entries` (no generic schema-agnostic writer); INSERT uses `Content-Profile: economy_map`; no UPDATE/DELETE helper is built (append-only stays trigger-enforced); service key is read from env, never logged |
| T-05-03 | Information Disclosure | processor agent key in proxy Authorization header | accept | Key already flows to the proxy for wallet summary (processor:6033) under the same `Authorization: Bearer` pattern; no new exposure introduced; key is not logged by the new helpers |
| T-05-04 | Spoofing / Bypass | classifier call routing | mitigate | Call goes through llm-proxy auth (per-agent key) per proxy.py:759-765; direct DeepSeek SDK path is explicitly NOT used (criterion 2), preventing an ungoverned/unaudited model call |
