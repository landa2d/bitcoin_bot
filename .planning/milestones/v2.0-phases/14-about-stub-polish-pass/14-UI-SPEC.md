>

## Spacing Scale

**Locked from Phase 11 — reused verbatim. Every value below is a multiple of 4px.** The token set is unchanged; POLISH-01 (D-04) does NOT add tokens — it **re-anchors loose/off-grid literals onto these existing tokens** and tightens vertical gaps within them.

| Token | Value | Usage |
|-------|-------|-------|
| `--space-xs` | 4px | Progress-dot gaps, agent-pill name→desc gap, inline icon gaps |
| `--space-sm` | 8px | Compact element spacing, brand gap, title→date gaps, pill grid label margin |
| `--space-md` | 16px | Default element gap; **agent-row grid gap**; card-internal gap; about paragraph bottom margin |
| `--space-lg` | 24px | Section side padding; card padding; agent-pill internal padding region |
| `--space-xl` | 32px | Section vertical breaks; section-label top margin |
| `--space-2xl` | 48px | Major section breaks |
| `--space-3xl` | 64px | Page-bottom breathing room |

### POLISH-01 vertical-rhythm sweep (D-04 — magnitude is this contract's call, token-anchored)

Direction (locked by D-04): **denser / more editorial, but minimalist-NOT-sparse — never cramped.** Magnitude is delegated to this UI-SPEC. The sweep snaps the live-cascade off-grid literals to the nearest token **one step tighter** where it reads loose, and confirms no off-grid spacing remains. Concrete re-anchors (executor target — all in `style-shared.css` unless noted):

| Surface | Current (off-grid / loose) | Target (token) | Rationale |
|---------|---------------------------|----------------|-----------|
| `#subscribe-section` padding | `40px 0` (line ~733) | `var(--space-xl) 0` → **32px** | 40px is between `--space-xl`/`--space-2xl`; step down to 32px tightens the largest loose block-break. |
| `.content-area` padding | `0 0 32px` + `padding-top: 20px` (lines ~130–134) | `0 0 var(--space-xl)` + `padding-top: var(--space-lg)` → bottom **32px**, top **24px** | 20px is off-grid → 24px (nearest token, still airy under the divider). |
| `.card` padding | `20px 20px 16px` (line ~258) | `var(--space-lg) var(--space-lg) var(--space-md)` → **24/24/16** | 20px off-grid → 24px; keeps card density consistent with section padding. |
| `.tier-label` margin | `32px 0 12px` (line ~223) | `var(--space-xl) 0 var(--space-sm)` → **32/0/8** | 12px label→grid gap off-grid → 8px tightens the group header. |
| `.section-label` family (mono labels) | bottom margins at `12px` literals (e.g. lines ~406, ~768, ~821) | `var(--space-sm)` → **8px** | Label-to-content gaps land on 8px consistently. |
| `.card` title `margin: 20px 0` / list `20px 0` (lines ~593, ~656) | `20px` literals | `var(--space-lg) 0` → **24px** | Off-grid → token; consistency over micro-tightening here (these are reading-view rhythms, keep generous). |

**Net-new About spacing (agent-pill block):** `.agent-row` grid `gap: var(--space-md)` (16px); `margin: var(--space-lg) 0` (24px above/below the row); `.about p` bottom margin `var(--space-md)` (16px); the eyebrow→title→page-sub→prose stack reuses the `.about-stub` rhythm already present (`padding-top: var(--space-2xl)`, `.page-title margin-top: var(--space-md)`).

**Hairline note:** existing `0.5px` borders (`.content-area`, `#subscribe-section`, `.bottom-bar`, subscribe input) are border *widths*, not spacing — exempt from the 4px rule by definition (same exemption Phase 11 took for `1px`). The sweep does NOT need to touch them; if the executor normalizes `0.5px`→`1px` for crispness it is optional and out of POLISH-01's required scope.

**Exceptions:** none — every spacing value lands on a 4px-grid token after the sweep. The chrome control paddings locked in Phase 11 (`8px 12px` tabs, `8px 16px` subscribe, `8px 20px` toggle, `12px` nav vertical) are already 4px-conformant and are NOT changed.
