| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-16-WS | Tampering / EoP | the loader's write surface | mitigate | The loader keeps a SINGLE narrow insert function that writes ONLY `block_body_versions` drafts — never `blocks`, never a generic schema-agnostic writer (T-07-WS purpose-scoped-writer pattern). | T-16-INJ | Tampering / Injection | body/frontmatter → PostgREST | mitigate | Body + frontmatter values are passed as JSON payload values via `httpx ... json=row` (parameterized) — never string-concatenated into SQL. PostgREST parameterizes the insert; there is no SQL surface in the loader. |
| T-16-FL | (fail-loud governance) | validate-all gate + non-2xx raise | mitigate | Missing/empty required field or a non-2xx response halts the whole batch loud (raise → `sys.exit(1)`) and lands nothing — no silent partial state (D-04/D-05).
