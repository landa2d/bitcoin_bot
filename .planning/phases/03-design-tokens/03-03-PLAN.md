<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Caddy → public visitors | `/style-map.css` and `/tokens-preview.html` become publicly reachable after rebuild |
| Docker compose host → web container | Rebuild changes the running container's `/srv` contents |
| `index.html` edit → all existing SPA users | One-line `<link>` insertion is loaded by every visitor of `/#/...` |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-08 | Tampering | `index.html` <link> insertion | mitigate | The new link is exactly one line, mirroring the existing `/style-shared.css` `<link>` syntax. Acceptance gates verify both the new line is present AND the existing line is still present (no accidental replacement). `git diff` would show a single-line addition. |
| T-03-09 | Denial of Service | Rebuild affects live site | mitigate | `docker compose up -d --build web` performs a rolling rebuild — Caddy briefly interrupts service for a few seconds. Acceptance gates verify `docker compose ps web` shows `Up` and that `/#/` (the existing SPA) still returns HTTP 200 after rebuild. If rebuild fails, the prior container remains running — no downtime risk beyond the rolling restart. |
| T-03-10 | Tampering | Public CSS file content | mitigate | The shipped `style-map.css` is reviewed via Plan 03-01 acceptance gates before this plan's rebuild. The acceptance gate `curl https://.../style-map.css | grep -F -- "--accent-teal-base:   #0F6E56"` confirms the live file matches the planned content (catches any accidental deploy of a stale or tampered file). |
| T-03-11 | Repudiation | Operator approval of visual checks | mitigate | The human-verify checkpoint records the operator's "approved" signal in the execution log. The numbered 9-step verification is explicit, so an issue can be pinpointed by step number rather than a vague "looks off". |
| T-03-SC | Tampering | No package installs in this plan | accept | No npm/pip/cargo installs; rebuild only re-runs the existing Dockerfile (Caddy base image, no new dependencies added). Package legitimacy gate not applicable. |
</threat_model>
