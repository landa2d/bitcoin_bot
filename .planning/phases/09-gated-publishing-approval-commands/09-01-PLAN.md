<objective>
Amend `economy_map.publish_block_version` (new migration 038) so that, on approval, the
parent block's `last_synthesized_at` advances from the approved draft's pinned
`synthesized_from_through` value — NOT `NOW()` (D-01, the IN-04 correctness contract).
The existing `NOW()` semantics (migration 033 §9, Step 4) double-count or skip timeline
entries created in the window between synthesis and operator approval, because the next
synthesis cycle's recency filter is `created_at > last_synthesized_at`.

This is a DB-only change: write the migration file, then APPLY it live via the Supabase
MCP `apply_migration` tool and confirm with drift-check. The live RPC must reflect the new
watermark semantics before Plan 09-02's command behavior can be verified (SC2) — a
container rebuild alone would pass against the OLD `NOW()` RPC, producing a false positive.

Purpose: Make the approval-time watermark exact so re-synthesis never loses or re-counts
window entries (GATE-02 correctness).
Output: `supabase/migrations/038_publish_block_version_watermark.sql`, applied live + drift-clean.
</objective>
