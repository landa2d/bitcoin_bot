<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| service_role → economy_map DB | All economy_map writes execute as service_role, which BYPASSES RLS by design — enforcement must be triggers + SECURITY DEFINER RPCs, never RLS (033:166-171, the historical failure actor) |
| RPC param → SQL | Operator-supplied slug/text/entry_id flows into RPC bodies as typed parameters |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-01 | Tampering | `timeline_entries` content columns | mitigate | Trigger exemption scoped to ONLY `reassigned_*` lifecycle columns; the re-emitted `timeline_entries_append_only()` still RAISEs on every content-column change (D-04). A too-broad exemption is the primary threat — content stays immutable. |
| T-10-02 | Elevation of Privilege | the four SECURITY DEFINER RPCs | mitigate | Each pins `SET search_path = economy_map, public` (T-02) and `REVOKE ALL FROM PUBLIC; GRANT EXECUTE TO service_role` — no PUBLIC execute, no search-path hijack. |
| T-10-03 | Tampering | `reassign_timeline_entry` source selection | mitigate | RPC gates the source SELECT on `block_slug = 'unsorted' AND reassigned_to_entry_id IS NULL` and RAISEs a typed exception otherwise — cannot re-file an already-filed/already-reassigned entry. |
| T-10-04 | Spoofing/Injection | RPC parameters | mitigate | All values passed as typed plpgsql parameters; NEVER string-interpolated into SQL. |
| T-10-05 | Denial of Service | concurrent open drafts | mitigate | `uq_block_body_versions_one_open_draft` partial UNIQUE index (D-07) is the structural backstop for the check-then-act race. |
| T-10-SC | Tampering | npm/pip/cargo installs | mitigate | No new package installs in this plan (pure SQL migrations) — N/A this plan; slopcheck applies in Plans 02/03 if any. |
</threat_model>
