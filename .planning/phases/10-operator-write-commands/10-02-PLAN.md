## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-06 | Elevation of Privilege | the four write commands | mitigate | Owner gate (`access_tier != "owner"` → refuse) is the FIRST statement in every handler, before any read or write (D-09). A non-owner cannot reach the RPC. This is the primary control for WRITE commands. |
| T-10-07 | Tampering/Injection | `_economy_map_rpc` fn interpolation | mitigate | `fn` is checked against `_ECONOMY_MAP_RPC_ALLOWLIST` BEFORE the URL is built; values pass as parameterized JSON (`json=params`), never string-interpolated into the URL/body. |
| T-10-08 | Tampering | slug input | mitigate | Every slug validated against `_ECONOMY_MAP_BLOCK_SLUGS` (seven-slug allowlist); `unsorted` and unknown slugs rejected with a typed error — input validation at the boundary. |
| T-10-09 | Spoofing | benign no-op on failure | mitigate | Non-2xx RPC RAISEs `RuntimeError` carrying `resp.text`; the dispatch try/except surfaces `Command failed: <e>` — a failed write never reads as a silent success (fail-loud, the "wallet bug" lesson). |
| T-10-10 | Tampering | open-draft invariant on /map-synth | mitigate | `handle_map_synth` reads the open-draft state and refuses to enqueue when one exists (D-02) — "ignore N/T" never bypasses the human-approval gate. |
| T-10-SC | Tampering | npm/pip installs | mitigate | No new package installs in this plan (uses existing `httpx`); slopcheck N/A. |
</threat_model>
