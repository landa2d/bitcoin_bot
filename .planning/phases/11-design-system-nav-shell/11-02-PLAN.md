<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → static assets (Caddy) | The only boundary; all assets static, served read-only. |
| user click → hash router (app.js) | Tab/back-control clicks change `window.location.hash`; getRoute() parses it. Hashes are not eval'd or interpolated into HTML — they select a view and a tab via string comparison only. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-03 | Tampering / XSS | setActiveTab() + new header markup | accept | setActiveTab toggles a CSS class and aria-current via classList/setAttribute — no innerHTML, no string-to-DOM, no hash interpolation into markup. The new header is static markup; tabs are fixed hash links. No new XSS sink beyond the existing codebase pattern. None/low. |
| T-11-04 | Spoofing | Subscribe button | accept | Reuses the existing scrollToSubscribe() verbatim (scrolls to an in-page anchor); introduces no new network call, endpoint, or credential path. No new surface. |
| T-11-SC | Tampering | npm/pip/cargo installs | accept | No package-manager installs (vanilla-JS SPA, no build step, no new dependency). Package legitimacy gate not applicable. |

No `high`-severity threats. The nav shell adds static markup, a class-toggling router hook, and CSS — no new user inputs, no auth, no backend/data-flow changes, no new external origin, no new innerHTML/inline-handler sinks beyond the existing pattern. CSP unchanged.
</threat_model>
