# Phase 11 — UI Design Contract

> Visual and interaction contract for the v2.0 Frontend Redesign foundation: the single light-mode CSS-variable palette, the Source Serif 4 / IBM Plex Mono typography system, and the persistent stateful 3-tab nav shell with back-arrow. Every later phase (12 Newsletter, 13 Agent Economy grid, 14 About + polish) restyles on top of this shell and inherits these tokens. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Stack note:** This is NOT a React/Tailwind/Next/shadcn project. It is a vanilla-JS single-page app — static `index.html` + hand-written CSS using CSS custom properties, served by Caddy as the `agentpulse-web` Docker container. Frontend lives entirely in `docker/web/site/`. The "design system" this phase establishes is a set of plain CSS variables, not a component-library theme. shadcn gate resolved to N. No `components.json` / `tailwind.config` / `package.json` exist (confirmed). Registry safety gate: **not applicable**.

## Design System

| Property | Value |
|----------|-------|

| Preset | not applicable |
| Component library | none |
| Icon library | none — use Unicode glyphs already in the codebase (`←` U+2190 for back, `·` U+00B7 as metadata separator, `↗` for source links). Do not add an icon dependency. |
| Font (reading/titles) | **Source Serif 4** (`'Source Serif 4', Georgia, serif`) |

### Font Loading (locked)

Use the Google Fonts stylesheet link, identical to the mockup:

```html
<link rel="preconnect" href="https://fonts.googleapis.com">
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
<link href="https://fonts.googleapis.com/css2?family=Source+Serif+4:ital,opsz,wght@0,8..60,400;0,8..60,500;0,8..60,600;0,8..60,700;1,8..60,400&family=IBM+Plex+Mono:wght@400;500;600&display=swap" rel="stylesheet">
```

**Rationale (load-bearing — do not change to self-hosting without a CSP edit):** The existing `docker/web/Caddyfile` Content-Security-Policy already whitelists `style-src ... https://fonts.googleapis.com` and `font-src https://fonts.gstatic.com`. Google Fonts therefore works with zero infra change. Self-hosting woff2 would require editing the CSP `font-src` to `'self'` and is explicitly out of scope for this UI pass. **Weights to load:** Source Serif 4 — 400, 500, 600, 700 (+ italic 400); IBM Plex Mono — 400, 500, 600.

## Spacing Scale

Source: mockup nav/section/card padding values + REDESIGN_BRIEF §6 ("tighten loose vertical gaps; minimalist but not sparse") + POLISH-01. Snapped to a 4px base. Where the mockup used a non-multiple-of-4 value (e.g. `14px` nav padding, `13px` tab padding, `18px` body margins), the contract uses the nearest 4px step; non-multiple values that are visually load-bearing in the mockup are listed as exceptions below.

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| `--space-xs` | 4px | Progress-dot gaps, inline icon gaps, toggle inner padding |
| `--space-sm` | 8px | Compact element spacing, brand gap, tab gap → title gaps |
| `--space-md` | 16px | Default element gap; grid gaps; nav side-padding step; card-internal gap |
| `--space-lg` | 24px | Section side padding (`main`/`.nav` horizontal padding), card padding |
| `--space-xl` | 32px | Block/section vertical breaks, section-label top margin |
| `--space-2xl` | 48px | Major section breaks (e.g. above About note, large eyebrow top margin) |
| `--space-3xl` | 64px | Page-bottom breathing room (`main` bottom padding region) |

Exceptions (intentional, from the mockup — these are chrome micro-paddings, not layout rhythm; keep as literal values, not tokens):
- Nav vertical padding `14px` and tab padding `7px 13px` — pill chrome sizing inherited from the mockup; acceptable non-token values.
- Subscribe button padding `9px 16px`, toggle button padding `7px 20px` — control chrome.
- Hairline borders `1px` (lines) — not spacing.

## Typography

Two families only. **Source Serif 4 = everything you READ (body paragraphs + every title/heading).** **IBM Plex Mono = UI chrome ONLY** (eyebrow/label, metadata like "Edition # · date", tab labels, brand wordmark, buttons, tags, code, section labels). **No monospace body paragraphs anywhere** (TYPE-01). A **single serif heading treatment** — the old second monospace heading style (`Courier New` uppercase accent `<h2>`/`<h3>`) is **removed** (TYPE-03).

Base body: **18px / line-height 1.62** serif (TYPE-03 target "~18px / ~1.6").

| Role | Family | Size | Weight | Line Height | Notes |
|------|--------|------|--------|-------------|-------|
| Body | Source Serif 4 | 18px | 400 | 1.62 | Reading text, article/about paragraphs. `body { font-size:18px; line-height:1.62 }`. |
| Page title (display) | Source Serif 4 | `clamp(30px, 5vw, 46px)` | 600 | 1.12 | Centered hero/section title; `letter-spacing:-.015em`. |
| Heading (in-content H2) | Source Serif 4 | 24px | 600–700 | 1.2 | Single serif heading style for article/block sub-sections (`margin` only varies). |
| List item title (edition / card title) | Source Serif 4 | 19–22px | 600 | 1.25 | Edition list `h3` 22px; economy card title 19px. `letter-spacing:-.01em`. |
| Eyebrow / kicker | IBM Plex Mono | 11.5px | 600 | — | UPPERCASE, `letter-spacing:.18–.22em`, accent or ink-faint. Chrome. |
| Metadata / page-sub | IBM Plex Mono | 13px | 400 | — | "Edition # · date", "synthesized … · block N of 8". `--ink-faint`. Chrome. |
| Tab label | IBM Plex Mono | 12.5px | 500 (600 active) | — | `letter-spacing:.02em`. Chrome. |
| Brand wordmark | IBM Plex Mono | 14px | 600 | — | `letter-spacing:.08em`, `--ink`. Chrome. |
| Button / Subscribe | IBM Plex Mono | 12px | 600 | — | `letter-spacing:.06em`, UPPERCASE label. Chrome. |
| Back-control | IBM Plex Mono | 12.5px | 500 | — | "← Back to [section]". `--ink-soft` → `--accent-ink` on hover. Chrome. |
| Section label (mono group label) | IBM Plex Mono | 11px | 600 | — | UPPERCASE, `letter-spacing:.18em`, `--ink-faint`. Chrome. |
| Code / inline `code` | IBM Plex Mono | 0.9em | 400 | — | Code only. Chrome. |

Weight policy: **two serif weights in active use** (400 body, 600 headings/titles; 700 reserved for the single article H2 emphasis, which is the same serif treatment scaled — not a new style). Mono uses 400 / 500 / 600.

## Color

A **single light-mode palette** as CSS variables, applied site-wide, **replacing the dark map theme** (COLOR-01). Hex values are locked verbatim from the REDESIGN_BRIEF §4 and mockup `:root`. **Exactly ONE accent** (violet) — no second brand color (COLOR-02). The old dual-accent system (technical teal `#00e5a0` / strategic `#7c3aed`) and the four-way tier-accent palette (teal/purple/coral/gray in `style-map.css`) are **retired** for this shell.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) — `--bg` | `#faf8f5` | Warm off-white page background; sticky-header translucent base. |
| Surface (30%) — `--surface` | `#ffffff` | Cards, agent pills, raised panels. |
| Ink (text) — `--ink` | `#1a1916` | Primary reading text + titles. |
| Ink soft — `--ink-soft` | `#55514a` | Secondary body, descriptions, back-control rest state. |
| Ink faint — `--ink-faint` | `#8a857c` | Metadata, page-sub, mono section/group labels, empty dots context. |
| Line — `--line` | `#e7e2da` | Hairline dividers, default card border, header bottom border. |
| Line strong — `--line-strong` | `#d8d2c7` | Empty progress-dot fill, toggle border, stronger separators. |
| Accent (10%) — `--accent` | `#5b3df5` | See reserved list. |
| Accent soft — `--accent-soft` | `#efeaff` | Active-tab background fill only. |
| Accent ink — `--accent-ink` | `#4a2fd6` | Accessible accent for text/links + active-tab text + hover-darkened accent. |
| Destructive | not applicable | No destructive actions in this phase (read-only public site). |

**Accent reserved for (explicit list — never "all interactive elements"):**
1. **Links** (anchor text in articles/about) — rendered in `--accent-ink` for contrast.
2. **The active nav tab** — `--accent-soft` background + `--accent-ink` text + `#ddd2ff` border.
3. **Card left-border accent** — 3px `--accent` stripe on economy-map cards (deepens to `--accent-ink` on hover). *(Establish the token here; the grid that consumes it is Phase 13.)*
4. **Progress dots (filled state)** — `--accent` fill on maturity/progress dots.
5. **Subscribe button** — solid `--accent` fill, white text, `--accent-ink` on hover.
6. **Eyebrow / kicker label** — accent-colored uppercase mono eyebrow.
7. **Brand dot** — the 9px `--accent` circle beside the wordmark.

Not allowed: any second hue, any per-tier accent color, the retired teal. Mode-driven accent swapping (the old technical/strategic palette flip) is gone — one accent in both newsletter modes.

**Contrast notes (accessibility):** `--accent-ink #4a2fd6` on `--bg #faf8f5` ≈ 7.0:1 (AA/AAA body text) — use it for link/active-tab text. `--accent #5b3df5` on `#fff` ≈ 5.4:1 — fine for the white-on-accent Subscribe button and as a non-text fill (borders/dots). `--ink #1a1916` on `--bg` ≈ 15:1. `--ink-faint #8a857c` on `--bg` ≈ 3.3:1 — **chrome/metadata only, never body copy** (consistent with TYPE-02 keeping metadata in mono chrome). All hex values are taken from the brief and treated as locked, not defaults.

## Nav Shell Contract (NAV-01..04)

The persistent shell is the load-bearing deliverable of this phase. Specifics:

**Structure (present on EVERY page / every hash route):**
- Sticky `<header>` (`position:sticky; top:0; z-index:50`) with translucent warm-white background `rgba(250,248,245,.86)` + `backdrop-filter: saturate(140%) blur(10px)` + `1px solid --line` bottom border. Keep the blur light.
- Left: **brand** — `--dot` (9px accent circle) + `AGENTPULSE` wordmark (IBM Plex Mono 14px/600). Clicking the brand returns to the Newsletter home (`#/`).
- Center: **three tabs** in exact order and copy: `Newsletter` · `Agent Economy` · `What is AgentPulse`.
- Right: **Subscribe** button (solid accent). Reuses the **existing subscribe flow** — it must invoke the current `scrollToSubscribe()` path (do not build a new subscribe mechanism; NAV-01).

**Active-state mechanism (NAV-02):**
- Active tab styling = `--accent-soft` bg + `--accent-ink` text + `#ddd2ff` border + weight 600 (class `.active` / `aria-current="page"`).
- Active state is derived from the **current hash route**, not from which link was clicked, so deep links and nested pages light the right tab. Mapping (hooks into the existing `getRoute()` in `app.js`):

  | Hash route | View | Active tab |
  |-----------|------|-----------|
  | `#/` (and empty) | edition list | **Newsletter** |
  | `#/edition/{n}` | single edition (nested) | **Newsletter** |
  | `#/map` | economy hub | **Agent Economy** |
  | `#/map/{slug}` | single block (nested) | **Agent Economy** |
  | `#/status` | maturity snapshot (nested under map) | **Agent Economy** |
  | `#/about` *(route added in Phase 14; tab ships now)* | about stub | **What is AgentPulse** |
  | `#/unsubscribe` | unsubscribe utility | none active (utility page) |

  Implement as a single function that maps route→section and toggles `.active` on the matching tab inside `route()` (called on load and on every `hashchange`). This mirrors the mockup's `parent` map but is driven by the real router.

**Back-control (NAV-03):**
- Every **nested** page shows a top-left back control: copy pattern **`← Back to [section]`**.
  - Single edition (`#/edition/{n}`) → `← Back to Newsletter`.
  - Single block (`#/map/{slug}`) and status (`#/status`) → `← Back to the map` (the Agent Economy section).
- Styling: IBM Plex Mono 12.5px, `--ink-soft` → `--accent-ink` on hover, no underline. (Replaces the current `← All editions` / `← Map` back-links.)

**One-click reachability + Map retirement (NAV-04):**
- From any section a reader reaches any other in one click via the always-present tabs.
- The old plain `<a class="nav-map-link">Map</a>` text link in `.nav-left` is **removed** — its destination is now the **Agent Economy** tab.
