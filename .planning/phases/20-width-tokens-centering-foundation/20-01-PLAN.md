<objective>
Establish the two coexisting, both-centered max-widths that kill the dead left gutter (WIDTH-01, D-06): a narrow prose axis (`--measure: 64ch`) for reading copy and a wide axis (`--wide: 1080px`) for tiled content, applied per-route. Widen the sticky nav onto the same `--wide` axis (D-02) so chrome and content edges line up, and retire the single 720px `.container` that crams every route into one too-narrow centered column.

Purpose: This is the structural foundation every later v2.2 visual phase (21-23) sits on. The D-06 root cause is already traced (RESEARCH §"D-06 Root Cause"): a single `<div class="container">` at `index.html:30` (max-width 720px, `style-shared.css:20`) wraps the hero, all six route-views, subscribe, and footer — it IS centered (`margin:0 auto`), so the "pinned center-left" cause is the non-obvious one the operator flagged. The wide map grid + lists crammed into a centered 720px column read as a dead gutter; nav (880px) vs content (720px) are two mismatched axes.

Output: `--measure`/`--wide`/`--gutter` tokens + `.prose`/`.wide` classes in `style-base.css :root`; the nav max-width widened to `--wide`; the legacy `.container` retired; `index.html` restructured so each route carries the correct axis; a small `renderHub` wrap so the map's prose intro stays narrow while its grid spans wide.
</objective>
