<objective>
Refactor `app.js`'s hash router into a two-mode model (`mode:'landing'|'detail'`) and restructure `index.html` so the four top-level sections (Newsletter list / Signals / Agent-Economy / About) render on ONE single-scroll landing page while editions (`#/edition/<n>`) and block pages (`#/map/<slug>`) stay deep-linkable detail routes. This is the SCROLL-01 navigation-architecture change — the structural foundation the Plan 02 scroll-spy + scroll-restore consume.

Purpose: SCROLL-01 converts the separate top-level routes into a single-scroll landing without losing the SEO/deep-linkability of editions + block pages. Per 21-RESEARCH.md the lowest-risk path is ADDITIVE: keep the existing `getRoute()`/`route()`/`showView()` shape, add a mode discriminator, and REUSE the existing `#list-view`/`#about-view`/`#map-view` DOMs as stacked `<section>` anchors (preserving every Phase 12-20 CSS selector + render function). The namespace split (slashed detail hashes tested FIRST, bare anchors resolve to landing) is the load-bearing complexity (21-RESEARCH.md Pitfall 1).

Output: bare-anchor nav + a `#landing` wrapper with 4 stacked sections + a static `#signals` shell in `index.html`; a two-mode `getRoute()`, a `route()` mode branch, a `showLanding()`/`showDetail()` split, detail-only `setActiveTab()`, decoupled loaders, and a `landingScrollY` module var in `app.js`. Scroll-spy, scroll-restore, smooth-scroll CSS, and the deploy/holistic-verify are Plan 02.
</objective>
