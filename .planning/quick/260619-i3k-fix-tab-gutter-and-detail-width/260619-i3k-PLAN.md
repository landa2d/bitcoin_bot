# Quick Task 260619-i3k: Tab gutter + detail-route width

## Problem

Two operator-reported live-site defects on `aiagentspulse.com`:

1. **No left gutter.** Text on the tabs/sections sits flush against the band edge. Root cause: `.content-area` (`style-shared.css:129`) uses the physical shorthand `padding: 0 0 var(--space-xl); padding-top: var(--space-lg)` which forces `padding-left/right: 0`. Because `style-shared.css` loads **after** `style-base.css`, that physical zero overrides the logical `padding-inline: var(--gutter)` set by `.wide`/`.prose`. Result: every `.content-area.*` wrapper loses its side gutter.

2. **Deep-dives & editions too narrow.** `#reader-view` (edition reader) and `#block-view` (agent-economy block deep-dive) are wrapped in `.content-area prose` (the 64ch `--measure` axis), narrower than the landing's `--wide` band. The operator wants them to match the main tab width.

## Tasks

### Task 1 — Restore the side gutter (`.content-area` → block-only padding)

**files:** `docker/web/site/style-shared.css`

**action:**
- At `:129`, replace the `.content-area` rule body so it sets only block (top/bottom) padding, leaving inline padding to `.wide`/`.prose`:
  - from: `padding: 0 0 var(--space-xl);` + `padding-top: var(--space-lg);` (keep `border-top: 1px solid var(--line-strong);`)
  - to: `padding-block: var(--space-lg) var(--space-xl);` + keep `border-top: 1px solid var(--line-strong);` (drop the now-redundant `padding-top` line)
- At `:1114` (mobile `@media (max-width:600px)`), change `.content-area { padding: 0 0 24px; padding-top: 16px; }` → `.content-area { padding-block: 16px 24px; }`

**verify:** `grep -nE "\.content-area\b" docker/web/site/style-shared.css` shows both rules using `padding-block` and no `padding:`/`padding-top:` resetting inline padding; `grep -c "padding-inline" docker/web/site/style-base.css` still 4 (untouched). No hex added.

**done:** `.content-area.wide`/`.content-area.prose` inherit `padding-inline: var(--gutter)` from `.wide`/`.prose` → gutter restored; the `#landing .prose` (style-base.css:347) and `#landing .wide .wide` (style-base.css:351) neutralizers still prevent double-guttering.

### Task 2 — Widen the detail routes to the `--wide` band

**files:** `docker/web/site/index.html`

**action:**
- `#reader-view` (`:164`): `<div class="content-area prose">` → `<div class="content-area wide">`
- `#block-view` (`:172`): `<div class="content-area prose">` → `<div class="content-area wide">`
- Leave `#status-view` (already `.content-area wide`) and the landing sections unchanged.

**verify:** `grep -nE "id=\"(reader|block|status)-view\"" -A1 docker/web/site/index.html` shows reader/block/status wrappers all `content-area wide`; no remaining `content-area prose` in index.html.

**done:** Editions + block deep-dives render on `--wide` (1080px) with the restored gutter (Task 1), matching the landing main tabs. app.js renders body innerHTML directly (no inner `.prose`/width wrapper) and there is no independent max-width on `#newsletter-content`/`#block-content`/`article`, so the wrapper class change is sufficient.
